I’ve had a SunPower system and 14, 400W solar panels installed on my house since May of 2018. It has always frustrated me that SunPower does not allow you to check the production and status of each panel out of the box. The app they give owners access to only displays the total amount of energy generated by the system over a given period of time. For most people this is probably fine and any addional displays would probably confuse the average owner. However, I (and probably you reading this) want to see all the data the system generates because I purchased the system and like hording data.

After some googling and reddit rabit holes, I came across the sunpower-pvs-exporter GitHub repo, and Scott Gruby’s fantastic blog post on how use a Raspberry Pi Zero (WH) as a bridge to call the PVS API.

System Overview

Identical to Scott Gruby setup, I used a RaspberyPi Zero (WH) and an Ethernet Adpater as a network bridge to allow me to call the PVS API from any client on my network. I then wrote a simple python script to ingest data from the PVS to InfluxDB every minute.
